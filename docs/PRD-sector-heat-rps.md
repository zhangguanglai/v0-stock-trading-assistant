## 2. 背景与目标

### 2.1 业务背景

当前股票列表仅显示行业名称，缺乏对板块整体强弱的可视化判断。投资者需要快速识别：
- 哪些板块是当前市场主线？
- 个股所属板块是否强势？
- 板块是否有资金关注？

### 2.2 目标

引入**RPS（Relative Price Strength，股价相对强度）**指标，为每只股票展示其所属板块的热度情况，帮助用户：
1. 快速识别强势板块
2. 判断个股是否处于热点板块
3. 辅助板块轮动策略决策

### 2.3 理论基础

**RPS由投资大师威廉·欧奈尔（William O'Neil）在《笑傲股市》中提出**

$$
\text{RPS} = \left(1 - \frac{\text{标的涨幅排名}}{\text{全市场统计标的总数}}\right) \times 100
$$

- 数值范围：0-100分
- RPS≥90：极强（前10%）
- RPS≥80：强势（前20%）
- RPS<50：弱势（后50%）

## 5. 算法规则

### 5.1 核心架构：双层指标体系

```
┌─────────────────────────────────────────────────────┐
│           最终热度 = RPS20 × 70% + 实时活跃度 × 30%    │
├─────────────────────┬───────────────────────────────┤
│     RPS20 (70%)     │      实时活跃度 (30%)          │
├─────────────────────┼───────────────────────────────┤
│ • 基于全市场28个行业 │ • 当日板块涨幅 (15分)         │
│   的20日涨幅排名     │ • 平均量比 (10分)             │
│ • 欧奈尔标准算法     │ • 换手率 (5分)                │
│ • 中期趋势强度       │                               │
└─────────────────────┴───────────────────────────────┘
```

### 5.2 RPS20 计算逻辑（基于全市场）

**数据源**: 调用 `/api/stock/industry-rps` 接口获取全市场所有申万一级行业的RPS20

**计算步骤**:

1. **获取全市场股票基本信息**
   - 调用 `tushare.stock_basic` 获取所有上市股票的行业分类
   - 按28个申万一级行业分组

2. **采样计算各行业20日涨幅**
   - 每个行业随机抽取最多10只代表性股票
   - 获取每只股票近20个交易日的K线数据
   - 计算: `(当前价 - 20日前收盘价) / 20日前收盘价 × 100%`
   - 取该行业样本股的平均值作为行业20日涨幅

3. **RPS20排名计算**
   ```typescript
   // 将所有行业按20日涨幅降序排列
   const sorted = allIndustries.sort((a, b) => b.change20d - a.change20d);
   
   // RPS20 = (1 - 排名位置 / 总行业数) × 100
   const rps20 = Math.round((1 - index / (totalIndustries - 1)) * 100);
   ```

**示例**: 种植业在28个行业中20日涨幅排名第3
$$
\text{RPS20} = \left(1 - \frac{3}{27}\right) \times 100 = 89\text{分}
$$

### 5.3 实时活跃度计算（权重30%）

| 维度 | 权重 | 分值范围 | 计算公式 | 说明 |
|------|------|---------|---------|------|
| **当日涨幅** | 15分 | 0-15 | `7.5 + (涨幅%/3) × 7.5` | 涨幅>3%=满分, 0%=7.5分, <-3%=0分 |
| **量比** | 10分 | 0-10 | `4 + (量比-1) × 6` | 量比>2=满分, 量比=1=4分, <0.8=2分 |
| **换手率** | 5分 | 0-5 | `1 + (换手率%/5) × 4` | >5%=满分, 2%=2分, <1%=1分 |

**换手率估算公式**:
```typescript
// 由于API未直接返回换手率，通过量比估算
const estimatedTurnover = avgVolume > 1 
  ? Math.min(avgVolume * 3, 15)   // 放量时换手率高(上限15%)
  : Math.max(avgVolume * 2, 0.5); // 缩量时换手率低(下限0.5%)
```

### 5.4 最终热度评分

```typescript
const heatScore = clamp(
  Math.round(rps20 * 0.7 + realtimeActivity),  // RPS20占70%, 活跃度30%

```
